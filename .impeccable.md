# FlexEnergy — Impeccable Design Context

## Design Context

### Users

End consumers managing prepaid energy contracts. **Mobile-first** — most check the app quickly on their phone when they get a low-balance alert or want to top up. Desktop use is occasional (periodic deeper review). The primary action loop is: check balance → top up → done. The secondary loop is: review usage → understand trends.

Users are non-technical. They do not want to feel like they are operating software — they want to feel like the energy company has their back. Confidence and clarity matter more than information density.

### Brand Personality

**Three words: Approachable · Assured · Bright**

- **Approachable**: Big controls, generous touch targets, warm light palette. Never intimidating or cold.
- **Assured**: Reliable data, confident hierarchy, numbers presented as fact not estimate. Design that feels like it will not break.
- **Bright**: The blue reads as trust and utility — keep it. Energy as a concept is light and motion; the UI should feel alive, not static.

Emotional goal: the user should feel reassured every time they open it. Not impressed by cleverness — *relieved* by clarity.

### Aesthetic Direction

- **Theme**: Light mode. White/pale surfaces with tinted neutrals leaning toward the brand blue.
- **Primary color**: Blue — not a generic corporate blue, but a confident, slightly warm blue. OKLCH-based palette.
- **Accent warmth**: Where possible, introduce a subtle warm tint (amber, amber-adjacent) to status and low-balance warnings — energy and heat are conceptually linked.
- **Typography**: Choose a display font with character for headings/large numbers (NOT Inter, NOT DM Sans). Pair with a clean, highly legible secondary face for body copy. Numbers must be large, bold, tabular.
- **Motion**: Fluid and present — spring transitions on navigation, staggered card entrances, number counters. Transitions should feel like a well-made iOS app, not a web form.
- **Controls**: Big. Minimum 48px touch targets. Buttons with generous padding. No tiny clickable text.
- **Anti-references**: NOT a console, NOT a telco portal, NOT a cold data dashboard, NOT clinical. Should feel closer to a well-designed fintech app (Monzo, Wise, Revolut) than a utility billing portal.

### Design Principles

1. **Numbers are heroes** — balance, kWh, percentages are the most important information. Make them impossible to miss: large type, strong contrast, centered in the visual field.
2. **Touch-first at every scale** — minimum 48px hit targets, 24px+ spacing between interactive elements. Desktop layouts gain density, never shrink the controls.
3. **Motion is communication** — every transition tells the user something. Navigation feels like physical movement between spaces. State changes (loading → loaded) are smooth reveals, not jarring replacements.
4. **One action per moment** — the primary CTA on every screen is unambiguous. Secondary actions are visually subordinate.
5. **React Native ready** — avoid CSS effects that won't survive a port (complex box-shadows, CSS transforms on scroll, filter-based effects). Prefer semantic structure + spacing over visual tricks.

### Tech Stack Context

React 19 + TypeScript, Tailwind v4, Motion (Framer Motion v12), Recharts, Radix UI primitives, Lucide icons. Demo app — all features functional, visual impact is a first-class requirement alongside working code.
